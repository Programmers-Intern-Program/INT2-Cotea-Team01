### 프로젝트 개요

- 프로젝트명
  - **코티(Cotea) - 코드티처 + 코치**
- 한 줄 소개 (서비스를 한 문장으로 설명)
  - 정답이 아닌 "방향성"을 제시해 스스로 사고하는 힘을 길러주는, 프로그래머스 코딩테스트 전용 AI 튜터 크롬 익스텐션
- 해결하고자 하는 문제 정의
  - 코딩테스트를 독학하는 사람들은 문제 풀이 중 막히면 검색을 통해 **정답 코드나 완성된 풀이**를 바로 접하게 되어, 스스로 사고하는 과정을 건너뛰게 되는 문제가 있음
  - 기존 AI 코딩 도우미들은 질문하면 바로 답을 알려주거나 전체 코드를 생성해줘서, 학습 효과보다는 "정답 복사"에 가까운 사용 패턴을 유도함
  - 반대로 힌트 없이 무작정 고민만 하면 비효율적인 시간 낭비로 이어지고, 학습 의욕이 떨어짐
  - 즉, **정답을 주지 않으면서도 사고를 진전시킬 수 있는 적정 수준의 가이드**가 부재함
- 타깃 사용자
  - 프로그래머스로 코딩테스트를 독학/준비하는 취업준비생, 학생
  - 알고리즘 풀이 경험이 적어 "접근 방법" 자체를 어려워하는 초·중급 학습자
  - 정답을 바로 보기보다 스스로 풀어내는 학습 경험을 선호하는 사용자
  - (지원 언어 한정) **Java**로 코딩테스트를 준비하는 사용자

### AI 활용 계획

- Claude API를 어떻게 활용할 것인지 구체적으로 기술
  - **단계별 개입 시점에 따라 다른 역할의 프롬프트를 호출**
    1. **문제 풀기 전**: 사용자의 접근 방식 이해도에 따라 4단계 힌트 중 하나를 제공
       - 1단계 (관점 힌트): 알고리즘명 공개 없이, 문제를 바라보는 키워드/관찰만 제공
       - 2단계 (접근 힌트): 알고리즘/자료구조 이름을 공개하고 왜 그 방향인지 간단히 설명
       - 3단계 (구현 힌트): 반복문/조건문/자료구조 사용 방식을 단계별로 안내, 의사코드까지 허용
       - 4단계 (코드 리뷰/디버깅): 사용자가 작성 중인 코드를 기준으로 문제 지점 분석, 필요시 2~3줄 부분 코드 제공
    2. **풀이 중 (막힘)**: 사용자의 현재 코드를 분석해 자료구조 추천, 상세 구현 힌트, 방향성 확인 등에 응답
    3. **오답/시간초과 시**: 어디가 왜 틀렸는지, 방향을 어떻게 바꿔야 하는지에 대한 진단 제공 (단, 질문하지 않으면 이유는 먼저 설명하지 않고 결과만 안내)
    4. **풀이 후**: 모범 답안 대비 코드 분석(효율성, 문제 의도 부합도), 유사 문제 추천, 개선 제안
  - 문제별 요구 풀이 방식, 난이도, 핵심 알고리즘 등은 **사전 전처리된 문제 메타데이터**를 프롬프트 컨텍스트로 함께 전달하여 정확도를 높임
  - 사용자의 현재 코드는 매 요청마다 함께 전달되어, AI가 "진행 상태"를 파악한 뒤 그에 맞는 응답을 생성
- 핵심 프롬프트 설계 방향
  - **대전제: 질문한 것에만 답한다** — 사용자가 요청하지 않은 정보(예: 오류 원인, 최적화 방법)는 먼저 제공하지 않음
  - **정답 유도 금지, 방향성 우선**: 완성된 코드나 알고리즘 전체를 생성하지 않고, 다음 사고 단계로 넘어갈 수 있는 힌트만 제공
  - **현재 작성 중인 코드의 완성을 최우선 목표로 설정**: 비효율적이거나 시간초과가 날 수 있는 코드라도, 알고리즘 자체가 틀리지 않았다면 일단 완성을 돕는 방향으로 가이드
  - **명백히 잘못된 접근(답이 나올 수 없는 알고리즘 선택)만 예외적으로 먼저 알림**
  - **기초 문법 오류는 자유 질의 응답으로 처리** (향후 별도 문법 검사기 도입 예정), 오류가 발생했는데 사용자가 물어보지 않으면 먼저 지적하지 않음
  - 시스템 프롬프트에 "힌트 단계", "문제 메타데이터", "질문 유형(버튼 선택 vs 자유 텍스트)"을 구조화하여 전달
- 예상 입출력 예시
  **예시 1 - 풀기 전 (1단계 힌트 요청)**
  - 입력: `[문제: 프로그래머스 Lv2 "타겟 넘버"] 사용자 질문: "완전 접근법이 안 떠올라요"`
  - 출력: "이 문제는 각 숫자마다 선택지가 두 가지씩 있고, 그 선택들이 순서대로 이어져요. 모든 경우를 하나씩 확인해봐야 하지 않을까요? 이런 구조를 볼 때 뭐가 떠오르시나요?”
  **예시 2 - 풀이 중 막힘 (자료구조 질문)**
  - 입력: `[현재 코드: BFS 미구현 상태] 사용자 질문: "방문 처리를 어떻게 해야할지 모르겠어요"`
  - 출력: "현재 좌표나 상태를 기준으로 방문 여부를 체크해야 하는데, 지금 코드에는 방문 처리 로직이 없네요. 방문 여부를 저장할 수 있는 자료구조를 하나 떠올려보실래요? (예: boolean 배열, Set 등)"
  **예시 3 - 오답 후 (방향성 질문)**
  - 입력: `[제출 결과: 시간초과] 사용자 질문: "왜 틀렸는지 알려주세요"`
  - 출력: "현재 코드는 시간초과가 발생했어요. 반복문 구조를 다시 살펴보시겠어요?" (사용자가 추가로 "이유가 뭔가요?"라고 물으면 그때 O(n²) 구조와 개선 방향을 상세 설명)

### 기술 스택

- 사용 언어 및 프레임워크
  - **프론트엔드**: Chrome Extension (JavaScript/TypeScript, HTML/CSS)
  - **백엔드**: Spring (Java)
  - **인프라**: AWS
  - **지원 대상 코딩 언어**: Java (1차 지원 언어 한정)
- 주요 라이브러리 또는 외부 API
  - **Anthropic Claude API**: 방향성 제시, 코드 분석, 힌트 생성 등 핵심 AI 기능
  - Spring Boot, Spring Web (REST API 서버)
  - AWS (EC2/서버 배포, RDS — 문제 메타데이터(580문제) 저장용 관계형 DB로 확정)
  - Chrome Extension Manifest V3 (익스텐션 권한/구조)
- 시스템 구성도 (간략하게)

```java
[프로그래머스 웹페이지]
│  (DOM에서 문제 정보 / 사용자 코드 수집)
▼
[Chrome Extension - Content Script/Popup]
│  (사용자 질문 + 코드 + 문제 메타데이터 전송)
▼
[Spring Boot 백엔드 서버] ── (RDS 조회: PROBLEM + 연관 테이블 JOIN)
│
▼
[Claude API] ── 방향성/힌트/분석 응답 생성
│
▼
[백엔드 서버] → [Chrome Extension] → 사용자에게 응답 표시
```

### 기능 목록

- 핵심 기능 (MVP 기준)
  1. 크롬 익스텐션을 통한 프로그래머스 문제 페이지 연동 (문제 정보 + 사용자 코드 수집)
  2. 챗봇형 UI: 자주 나오는 질문은 버튼으로 제공 + 자유 텍스트 입력 지원
  3. **풀기 전** 단계별 힌트 제공 (1~4단계)
  4. **풀이 중** 막힘 대응: 자료구조 추천, 구현 힌트, 방향성 확인
  5. **오답 시** 진단: 어디가/왜 틀렸는지, 방향 전환 힌트 (질문 기반, 선요청 후설명 원칙)
  6. 문제별 요구 풀이 방식 데이터 사전 전처리 및 활용
  7. Java 언어 코드 진행 상태 파악 로직
- 추가 기능 (여유 시 구현)
  - **풀이 후** 코드 분석: 모범 답안 대비 최적화/문제 의도 부합도 평가
  - 유사 문제 추천
  - 문법 오류 자동 진단 (별도 문법 검사기)
  - 사용 횟수 제한 기능 (예: 문제당 3회 또는 시간당 5회 – 무료/체험판 정책)
  - 풀이 결과 리포트 생성
  - 코드에 주석을 달아 질문하는 방식 지원
